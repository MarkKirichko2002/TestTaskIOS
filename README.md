Test-Task-iOS

Архитектура

Проект построен на архитектуре MVVM (Model-View-ViewModel) для обеспечения четкого разделения ответственности и упрощения тестирования.

- Model (Модель):
  - Post/PostModel: Представляет структуру данных поста, полученную от API и сохраняемую в базе данных.

- View (Представление):
  - PostsListTableViewController: Главный ViewController, отвечающий за отображение списка постов.
  - PostTableViewCell: Кастомная ячейка таблицы для отображения отдельного поста.
  - LaunchScreen.storyboard: Экран запуска приложения.

- ViewModel:
  - PostsListViewModel: Содержит бизнес-логику экрана списка постов. Получает данные от сервисов, преобразует их в удобный для View формат и уведомляет View о необходимости обновления через замыкания или наблюдаемые свойства.

- Services (Сервисы):
  - API: Слой для сетевых запросов.
  - APIService: Отвечает за загрузку постов из сети.
  - DB: Слой для работы с локальной базой данных.
    - DBManager: Отвечает за сохранение, извлечение и удаление постов в/из базы данных.
  - Network:
    - NetworkManager: Утилитарный класс для проверки соединения с интернетом.

- Animation:
  - AnimationClass: Вспомогательный класс для применения анимаций к UI-элементам.

- Supporting:
  - AppDelegate, SceneDelegate: Стандартные классы жизненного цикла приложения.

Поток данных:
1. View сообщает ViewModel о действии пользователя (например, viewDidLoad)
2. ViewModel запрашивает данные у APIService (при наличии сети) или у DBManager (оффлайн-режим)
3. Сервисы возвращают данные ViewModel
4. ViewModel обрабатывает и преобразует данные в массив готовых к отображению моделей
5. ViewModel уведомляет View через binding о том, что данные обновились
6. View получает обновленные данные от ViewModel и перезагружает пользовательский интерфейс

 Список использованных технологий

- UIKit - основной фреймворк для построения пользовательского интерфейса
- Foundation - базовый фреймворк
- Network - для выполнения сетевых запросов и мониторинга сети
- Core Data - для локального кэширования данных и работы в оффлайн-режиме
- Архитектура: MVVM
- Менеджер зависимостей: Используется встроенная система управления зависимостями

 Инструкция по сборке

1. Клонируйте репозиторий:
bash
git clone [https://github.com/MarkKirichko2002/TestTaskIOS]
cd Test-Task-iOS

2. Откройте проект в Xcode:
Откройте файл Test-Task-iOS.xcodeproj в Xcode.

3. Выберите целевое устройство:
Выберите симулятор или подключенное устройство в верхней части окна Xcode.

4. Соберите и запустите проект:
Нажмите Cmd + R или кнопку "Run" в Xcode.

Приложение должно успешно скомпилироваться, запуститься и отобразить список постов.
